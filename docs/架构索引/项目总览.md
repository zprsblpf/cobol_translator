## 1. 项目是什么

把单个大型 COBOL 程序（`ZPOLDWNM.cob`）翻译为 Java。两条主线：

- **离线拆解线（确定性，无 LLM）**：`scripts/decompose.py` → `decompose/` 包，把整份 .cob 按规则切成
  独立小块，写文件 + 生成 `manifest.json`，可选幂等入库 MongoDB（`db/`）。供后续逐块翻译消费。
- **翻译线（LangGraph + LLM 编排）**：`main.py` → `graph/` 编排 parse → analyze → translate → validate → assemble，
  调用 `parser/ analyzer/ translator/ validator/`，LLM 配置集中在 `config/`。

> 当前活跃工作在「拆解线」（步骤01 拆解、步骤02 入库实现+包化拆分）。翻译线为既有骨架。

## 2. 整体架构图

```
                          ┌─────────────────────────── 拆解线（确定性） ───────────────────────────┐
  ZPOLDWNM.cob ──► scripts/decompose.py (瘦入口)
                          │   ├─► decompose/core.decompose()
                          │   │       ├─► decompose/structure  ─┐
                          │   │       ├─► decompose/blocks       ├─(用)─► decompose/lines ─► decompose/models
                          │   │       └─► decompose/manifest  ──┘     └─► <out>/拆解/*.cob + <out>/manifest.json|md
                          │   └─► decompose/importer.import_mongo() ─► db/mongo_client.MongoStore ─► MongoDB
                          └──────────────────────────────────────────────────────────────────────┘

                          ┌─────────────────────────── 翻译线（LLM 编排） ───────────────────────────┐
  main.py (CLI) ──► graph/graph.build_translation_graph()  [LangGraph]
                          │   state: graph/state.py
                          │   nodes: graph/nodes.py
                          │     parse_cobol_node    ─► parser/cobol_parser, parser/copybook_parser
                          │     semantic_analysis   ─► analyzer/semantic_analyzer
                          │     translate_node      ─► translator/llm_translator ─► translator/prompt_builder ─► config/llm_config
                          │     validate_node       ─► validator/
                          │     assemble_node       ─► output/
                          └──────────────────────────────────────────────────────────────────────┘
  公共：log_utils.py（日志）、knowledge/（映射与 copybook 注册）、config/（LLM 配置）
```

## 3. 调用关系（依赖方向，单向无环）

**拆解线**
- `scripts/decompose.py`（入口）→ `decompose.core`、`decompose.importer`
- `decompose.core` → `decompose.structure` / `decompose.blocks` / `decompose.manifest`
- `decompose.structure` → `decompose.lines`
- `decompose.blocks` → `decompose.lines` / `decompose.models`
- `decompose.manifest` → `decompose.blocks`(slug_filename)
- `decompose.importer` → `db.mongo_client.MongoStore`
- `scripts/verify_decompose.py`（独立校验，不被上述调用）→ 读 `<out>/manifest.json` 与 `<out>/拆解/*.cob`

**翻译线**
- `main.py` → `graph.graph` → `graph.nodes` →（各节点）`parser/ analyzer/ translator/ validator/`
- `graph.nodes` / 各节点 → `graph.state`（状态）、`log_utils`（日志）
- `translator.llm_translator` → `translator.prompt_builder` → `config.llm_config`

**WSAA 块翻译线（确定性，无 LLM；步骤03）**
- `scripts/translate_wsaa.py`（入口）→ `parser.ws.parse_ws` → `translator.wsaa.render_wsaa`
- `parser.ws`：`lines` →（`entry`/`conditions`/`value`/`pic`）→ `tree` → `model`
- `translator.wsaa`：`render_class` →（`render_field`/`render_condition`/`render_view`）
- `render_field` → `parser.variable_resolver._cobol_to_java_name`（复用）；
  `render_class` → `translator.skeleton._class_base`（复用）；`parser.ws.pic` → `config/type_mappings.yaml`

## 5. 每个包 / 类的功能简述

- **decompose/**（拆解功能包，纯逻辑无 argparse）：`models` 定义 `Block`；`lines` 做列区清洗与停用/注释判定；
  `structure` 定位 DIVISION/COPY 并提取入参；`blocks` 切 SECTION 并构 WS/META 块；`manifest` 落盘切片与清单；
  `importer` 幂等入库；`core` 串起全流程。逐块职责单一、依赖单向。
- **db/**（数据访问层）：`MongoStore` 是 MongoDB 薄封装，pymongo 惰性导入，提供
  `ensure_unique_index/upsert(三态)/count/close`，幂等键 `(program, block_name)`，`$setOnInsert` 保护回填字段。
- **graph/**（LangGraph 编排）：`state` 持有翻译过程状态；`nodes` 是各阶段节点；`graph` 把节点连成有向工作流。
- **parser/**：`CobolParser` 解析主程序结构，`CobolCopybookParser` 解析 copybook。
- **analyzer/**：`SemanticAnalyzer` 提取业务语义，辅助高质量翻译。
- **translator/**：`PromptBuilder` 造提示词，`LLMTranslator` 调用 LLM 产出 Java。
- **validator/**：校验生成的 Java（骨架）。
- **config/**：`llm_config` 集中管理 LLM 连接与参数；`type_mappings.yaml` 驱动 PIC→Java 类型。
- **parser.ws/**（WSAA 解析）：把 WORKING-STORAGE 块解析为 `WsNode` 森林（类型/宽度/88/REDEFINES 已回填）。
- **translator.wsaa/**（WSAA 渲染）：森林 → Java 类，叶子→字段、88→布尔方法、重叠组/REDEFINES→派生视图。
- **根**：`main.py` CLI 驱动翻译线；`log_utils.py` 公共日志。
