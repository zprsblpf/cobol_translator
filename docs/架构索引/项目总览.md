## 1. 项目是什么

把单个大型 COBOL 程序（`ZPOLDWNM.cob`）翻译为 Java。核心是**本地确定性翻译**：

- **翻译主线（本地确定性，config 标准驱动，无 LLM）**：`config/*.yaml`（翻译标准正本，经
  `config/spec_loader` 访问）落地成本地翻译标准 → `parser/`（本地化解析：整程序 `cobol_parser`、
  工作存储 `ws/`）→ `translator/`（落地渲染：数据块 `wsaa/`、主类骨架 `skeleton_gen/`、语句级规则引擎
  `rules`）→ Java 产物（`assemble` 组装、`validator` 校验）。`analyzer/` 提供确定性调用链/数据流支撑。
- **AI 辅助层（`graph/`，LangGraph）**：编排上述确定性翻译，仅对规则兜不住的叶子语句调 LLM 兜底
  （+ Chroma RAG）。是开发期 AI 辅助，非生产主路径。
- **离线拆解线（确定性，无 LLM）**：`scripts/decompose.py` → `decompose/` 包，把整份 .cob 按规则切成
  独立小块，写文件 + 生成 `manifest.json`，可选幂等入库 MongoDB（`db/`）。供逐块翻译消费。

## 3. 调用关系（依赖方向，单向无环）

**拆解线**
- `scripts/decompose.py`（入口）→ `decompose.core`、`decompose.importer`
- `decompose.core` → `decompose.structure` / `decompose.blocks` / `decompose.manifest`
- `decompose.structure` → `decompose.lines`
- `decompose.blocks` → `decompose.lines` / `decompose.models`
- `decompose.manifest` → `decompose.blocks`(slug_filename)
- `decompose.importer` → `db.mongo_client.MongoStore`
- `scripts/verify_decompose.py`（独立校验，不被上述调用）→ 读 `<out>/manifest.json` 与 `<out>/拆解/*.cob`

**翻译主线（本地确定性，config 标准驱动）**
- `config.spec_loader` → `config/*.yaml`（标准正本）；下方各渲染器读它落地本地标准
- `parser.cobol_parser.parse` → CobolProgram；`parser.ws.parse_ws` → WsNode 森林；`parser.variable_resolver` → Java 字段
- `translator.wsaa` / `translator.skeleton_gen` / `translator.rules`(+`segmenter`) → Java（详见下方各产线）
- `analyzer.callgraph` / `analyzer.dataflow` → 确定性调用链/数据流（`dataflow` → `parser.variable_resolver`）

**AI 辅助层 graph/（LangGraph 编排 + 叶子兜底）**
- `main.py` → `graph.build_graph` → 4 节点：`parse_cobol`→`build_context`→`translate_section`(Send 并行×4)→`assemble`
- `parse_cobol_node` → `parser.pipeline.build_parse_result`（= `cobol_parser.parse` + `variable_resolver.resolve`）
- `build_context_and_skeleton_node` → `translator.context`（用 `analyzer.callgraph/dataflow` + `translator.naming/skeleton`）
- `translate_section_node` → `translator.segmenter` + `translator.rules`（确定性）；兜不住的叶子 → LLM + Chroma RAG → `translator.postprocess`
- `assemble_node` → `translator.assemble`（用 `translator.skeleton`）
- 各节点 → `graph.state.TranslationState`、`log_utils`

**WSAA 块翻译线（确定性，无 LLM；步骤03，步骤05 改造为规范驱动）**
- `scripts/translate_wsaa.py`（入口）→ `parser.ws.parse_ws` → `translator.wsaa.render_wsaa`
- `parser.ws`：`lines` →（`entry`/`conditions`/`value`/`pic`）→ `tree` → `model`
- `translator.wsaa`：`render_class` →（`render_field`/`render_condition`/`render_view`）；`render_view` → `storage`（定宽存储串）
- `render_field`/`render_class` → `config.spec_loader`（类型/初值/命名/样式串，步骤05 改造；产物回归不变）

**骨架翻译线（确定性，无 LLM；步骤05 骨架 + 步骤07 方法体）**
- `scripts/translate_skeleton.py`（入口）→ `parser.cobol_parser.parse` → `translator.skeleton_gen.render_skeleton`
- `translator.skeleton_gen`：`program_model`（CobolProgram→骨架要素）→ `copy_resolver`（COPY 角色分流）→ `render_skeleton`（装配主类）
- `skeleton_gen` → `config.spec_loader`（类名/COPY 角色/实体类）+ 复用 `translator.skeleton._section_to_method`（段名→方法名）
- 步骤07：`render_skeleton` → `body_context`（`build_body_ctx` 备料 → `segmenter`+`rules` 译方法体 → 瘦后处理）；
  方法体不再空体，规则兜不住的叶子落 `// TODO 叶子待译`（无 LLM）。复用 `parser.variable_resolver` + `translator.naming` + `translator.postprocess`
- 步骤13：paragraph 级 `PERFORM A THRU B` → `body_context._build_proc_order`（建四元过程单元顺序表注入 `Ctx`）；
  `rules._perform_range` 两级查找（SECTION 级零回归 / paragraph 级 `_perform_range_paragraph` 解析区间→登记 `Ctx.pending_range_methods`→返回 `this.aThruB();`，命名经 `spec_loader.perform_range_method`）；
  `render_skeleton` 收尾 drain pending → `body_context.render_pending_range_methods` 译合成区间方法 → 发射为类级方法
- 步骤14：THRU 区间内 GO TO 交互 → `_perform_range_paragraph` 登记改存**带标签**单元序列 `[(label, body),…]`（不丢标签）；
  drain 经 `body_context.translate_paragraphs_body`（跳过 `split_paragraphs` 有损往返，标签直传；`translate_section_body` 重构为其薄封装）→
  `rules.build_section(force_sm=True)` 重建区间内 paragraph 状态机：区间内 GO TO 回跳=循环 / 前向=精确跳转 / 出口=return / 区间外=保守 TODO-GOTO（`force_sm` 仅作用合成区间方法，普通 SECTION 渲染零变化）
- 步骤15（C-3）：单条 `PERFORM <paragraph>`（无 THRU）→ `rules._perform_single_paragraph`（`_perform_range` 无 THRU 分支改调）：
  SECTION→维持 `this.<sec>()`；paragraph（proc_order 恰一次、非 SECTION、合成名不撞 SECTION 方法）→登记单单元 pending `[(X, body)]`、
  调用点 `this.pXxx();` 不变（渲染期补出 `void pXxx(){…}`，B1 补实参）；兜不住→`this.pXxx()` + 可见 TODO（消除静默坏调用）。复用步骤13/14 渲染/命名机制
- 步骤16：PERFORM 循环复杂变体 → `rules._perform_loop`（`_sk_perform` 抽出，§17 瘦身）：`WITH TEST AFTER`（仅无 VARYING）→ `do{…}while(!(cond));`；
  `VARYING…AFTER…` → `_parse_varying_clauses` 切子句序列 → 嵌套 `for`（任意层）；UNTIL/TIMES 基本形不变；任一子句兜不住 / VARYING+TEST AFTER → 整条落 LLM 叶子（all-or-nothing）

**相2 ASG 旁路线（步骤17；自研轻量 ASG，旁路并存不接旧路径）**
- `asg.build_asg(program)`（瘦门面，唯一入口）→ `asg.builder.build` → `asg.registry.build`（复用 `body_context._build_proc_order` 四元 → `ProcUnit`；`program.working_storage+linkage_vars` → `SymbolTable`）
- `asg.builder._lift`（**唯一 token 嗅探点**）只读复用 `translator.segmenter.segment`/`split_paragraphs` 取确定性切分树 → 提升为带类型节点（`IfStmt/PerformStmt/GotoStmt/CallStmt/Leaf` 等），引用经 `ProcRegistry.resolve`/`resolve_thru` 解析；PERFORM 头部修饰词读 `config.grammar_loader.perform_keywords()`
- `asg.builder._lift` 提升的语句类型含 `MoveStmt`（步骤18 项3①：MOVE 自 `Leaf` 兜底升为带类型节点）
- `asg.visitor.AsgVisitor`（`visit_<类名>` 分派）+ 单点 demo `GotoJavaVisitor`（GO TO→Java）+ `LeafJavaVisitor`（`visit_MoveStmt`→`translate_move`；`visit_IfStmt`→`translate_condition` 渲染 `if(){…}else{…}`；`visit_PerformStmt`→`translate_perform_loop` 渲染 while/for/do-while 循环壳；`visit_CallStmt`→`translate_call` 固化散点 CALL（matched=False→`// TODO-CALL`）；`visit_Leaf`→`translate_arith_assign` 固化 7 算术/赋值动词，兜不住才占位（步骤22），与旧 rules 同函数）
- **绞杀项3①（步骤18）**：rules 的 MOVE 译器 + 表达式底座下沉为公用 `translator/leaf/`，rules 委托、visitor 共调（两路同函数同 ctx→产物逐字符一致）；`scripts/diff_asg_vs_legacy.py` 作每刀比对闸
- **绞杀项3②（步骤19）**：rules 的条件翻译 `_try_condition` 下沉为 `translator/leaf/cond.py`，rules 委托（别名导回，4 处调用点零改）、`visit_IfStmt` 共调；比对闸扩 `--verb IF`（比对条件串）
- **绞杀项3③（步骤20）**：rules 的 PERFORM **循环子句**翻译 `_perform_loop` 三件套下沉为 `translator/leaf/loop.py`，rules 委托（原名导回）、`visit_PerformStmt` 共调；比对闸扩 `--verb PERFORM`（比对循环壳）。**只切①循环子句**；②THRU区间/目标解析、③BEGN foreach+struct_rebind（依赖 `proc_order`/`pending_range_methods` 骨架态）留后续「骨架装配迁 visitor」刀
- **绞杀项3④（步骤21）**：rules 的**散点 CALL 兜底**翻译 `_t_call`（+ IO 映射解析 `resolve_io_info`/`derive_io_info`）下沉为 `translator/leaf/call.py`，rules 委托（别名 `_t_call` + 再导入 `resolve_io_info`/`derive_io_info` 保 `rules.*` 公开面，结构吸收 4 处 + 外部引用零改）、`visit_CallStmt` 共调；比对闸扩 `--verb CALL`（比对 `(lines,matched)`）。**只切①散点兜底**；②setup+CALL+IF 结构吸收、③struct_rebind 留后续刀。`LeafCtx` 首次扩 4 字段（io_programs/io_default_pattern/date_programs/system_programs）；`CallStmt` 加 `tokens` 字段、`_lift` 填充（供 `translate_call` 解析）。其余动词（STRING…）仍在 rules，旧路径仍是产物正本（未下线）
- **绞杀项3⑤（步骤22）**：rules 的 **7 算术/赋值叶子译器**（INITIALIZE/SET → `leaf/assign.py`；ADD/SUBTRACT/MULTIPLY/DIVIDE/COMPUTE + `_arith_val` → `leaf/arith.py`）下沉为公用底座，rules 委托（再导入 7 名，`_dispatch_leaf` 零改）、`visit_Leaf` 经统一入口 `translate_arith_assign`（依次试 `translate_assign`→`translate_arith`，verb 集互斥）共调；比对闸扩 `--verb ARITH`（比对 `(lines,matched)`）。**不新增节点**（7 类已落 `Leaf`，单点委托）；占位单调收敛（只减不增）。STRING/UNSTRING/INSPECT 等仍交 LLM

- **绞杀项3⑥（步骤23）**：rules 骨架层 `_sk_control` 的 **flow_label-无关分支**（GO/GOBACK/STOP/EXIT/CONTINUE/NEXT）+ `_sk_evaluate` 的 **switch 壳判定**下沉为公用 `translator/leaf/control.py`（`translate_control`/`translate_evaluate`/`evaluate_case_label`），rules 委托（dispatch 分支保留）、`visit_EvaluateStmt`(新增)/`visit_GotoStmt`(改委托)/`visit_Leaf`(兜底再试) 共调；比对闸扩 `--verb CONTROL`。`GotoStmt` 加 `tokens` 字段、`LeafCtx` 扩 `known_sections`/`section_to_method` 2 字段；**`GotoJavaVisitor` 步骤17 demo 退役并入 `LeafJavaVisitor`**。**dispatch 模式状态机**（`__pc`/`continue FLOW`，依赖 `flow_label`/`flow_paragraphs` 骨架态）守界留**骨架装配迁 visitor 刀**
- **骨架装配迁 visitor（步骤24）**：新增 ASG 侧 `SectionJavaVisitor`，先把 `build_section` 的 paragraph 装配、状态机壳、flow dispatch、PERFORM out-of-line 目标解析迁入 visitor，并扩比对闸 `--verb SECTION`；后续步骤25-29 已补齐结构吸收并切入主线。
- **BEGN/NEXTR 结构吸收迁 visitor（步骤25）**：新增 ASG 结构节点 `BegnForeachStmt` 与 `structure_rewrite` pass，把旧 `_rewrite_begn_loops` Pass 1（BEGN+NEXTR 自跳循环 → List+for-each）迁入 `SectionJavaVisitor`，并补齐 visitor 侧 `struct_rebind` 语义；步骤29 后随 `SectionJavaVisitor` 进入主线。
- **单次 BEGN 结构吸收迁 visitor（步骤26）**：新增 `BegnSingleStmt` 与 `rewrite_begn_single`，迁旧 `_match_begn_single/_render_begn_single`（setup+CALL+IF → `findBy...Begn` + `list.isEmpty()`）；步骤29 后随 `SectionJavaVisitor` 进入主线。
- **READR/READS 单条读结构吸收迁 visitor（步骤27）**：新增 `IoReadSingleStmt` 与 `rewrite_readr_single`，迁旧 `_match_readr_single/_render_readr_single`（setup+CALL+[STATUZ IF] → finder + null 判断/try-catch）；步骤29 后随 `SectionJavaVisitor` 进入主线。
- **rules.build_section 旧路降级（步骤30）**：`rules.build_section` 不再作为主线 SECTION 正本，只保留为 ASG 异常 fallback 与 `diff_asg_vs_legacy.py` reference；主线默认入口保持 `body_context._translate_paragraphs_body_asg`。

**相3 叶子翻译公用底座（步骤18/19/20/21/22/23 绞杀项3①②③④⑤⑥；rules 与 asg.visitor 共用）**
- `translator.leaf.translate_move(toks, ctx)`（迁自 `rules._t_move`）+ `translate_condition(toks, ctx)`（迁自 `rules._try_condition`）+ `translate_perform_loop(header, ctx, indent)`（迁自 `rules._perform_loop`）+ `translate_call(toks, ctx)`（迁自 `rules._t_call`，附 `resolve_io_info`/`derive_io_info`）+ `translate_assign`/`translate_arith`/`translate_arith_assign`（迁自 rules 7 算术/赋值译器，步骤22）+ `expr` 表达式底座（operand/assign/struct/判型，迁自 rules）+ `LeafCtx` 窄契约
- ASG 侧 leaf-ctx 直接复用 `body_context.build_body_ctx`（与旧路径同一 `Ctx`）；依赖单向 `rules → leaf`、`asg.visitor → leaf`，无环

**规范访问层（步骤05；两条确定性产线共享）**
- `config.spec_loader` → `config/*.yaml`（type_mappings/naming_conventions/copy_mappings/wsaa_translation_spec）
  + 复用 `parser.variable_resolver._cobol_to_java_name`、`parser.ws.value.java_init`

**三相演进旁路（相1 预处理 + 相2 ASG；步骤15/16/17，与旧路径正交、单向无环）**
- 相1：`preprocess.preprocess_file` → `preprocess.line_stream`(`build`/`strip_line`) → `preprocess.columns` + `preprocess.dialect`（读 `config.grammar_loader`）；旧 `parser.cobol_columns` 薄重导出相1
- 相2：`asg.build_asg(program)` → `asg.registry.build`（复用 `skeleton_gen.body_context._build_proc_order`）+ `asg.builder.build`（只读复用 `translator.segmenter.segment/split_paragraphs`、`config.grammar_loader.perform_keywords`）→ `asg.nodes` 节点树
- 单点验证：`asg.visitor.GotoJavaVisitor.visit(GotoStmt)`（自证逐字符等于 `translator.rules._sk_control`，但**不 import rules**）
- **旁路约束**：asg 不被旧 `build_skeleton`/`assemble`/`graph` 调用，零改旧 `parser/segmenter/rules`（相3 逐类迁 visitor 后才接线）

## 5. 每个包 / 类的功能简述

- **decompose/**（拆解功能包，纯逻辑无 argparse）：`models` 定义 `Block`；`lines` 做列区清洗与停用/注释判定；
  `structure` 定位 DIVISION/COPY 并提取入参；`blocks` 切 SECTION 并构 WS/META 块；`manifest` 落盘切片与清单；
  `importer` 幂等入库；`core` 串起全流程。逐块职责单一、依赖单向。
- **db/**（数据访问层）：`MongoStore` 是 MongoDB 薄封装，pymongo 惰性导入，提供
  `ensure_unique_index/upsert(三态)/count/close`，幂等键 `(program, block_name)`，`$setOnInsert` 保护回填字段。
- **graph/**（AI 辅助编排层）：`state` 持翻译状态；`nodes` 4 节点编排确定性翻译、仅对兜不住的叶子调 LLM；`graph` 连成工作流（Send 并行派发 SECTION）。非生产主路径。
- **preprocess/**（相1 预处理，步骤15/16）：三相架构第一相。`columns` 定列格式行级处理单一正本（停用/注释/调试判定+列区清洗）；`line_stream` 产「干净行流」`CleanSource`（带原始行号回溯）；`dialect` 方言归一（GO→GO TO 等，规则读 config）；`__init__` 瘦门面 `preprocess_file`。旧 `parser/cobol_columns` 薄重导出于此；旧 `cobol_parser` 经薄委托消费。后续相2/相3 详见 `架构演进-三相分层…初步设计.md`。
- **asg/**（相2 自研轻量 ASG，步骤17；三相架构第二相，旁路并存）：把 `segmenter` 的裸 `Stmt` token 袋**提升**为带类型节点树（`nodes`）并解析过程/数据引用（`registry`：`ProcRegistry.resolve/resolve_thru` + `SymbolTable`）。`builder._lift` 是**唯一 token 嗅探点**（收口一处），`visitor` 以 `visit_<类名>` 分派、单点 demo `GotoJavaVisitor` 自证「访问类型节点」可替代 rules 的 token 嗅探。**零改旧 parser/segmenter/rules、不接主翻译**——绞杀分期②第一刀，旧路径回归闸不动。后续相3 逐类迁 visitor 见 `架构演进-三相分层…初步设计.md` §5。
- **parser/**（本地化解析，主线）：`cobol_parser.parse` 解析整程序结构，`variable_resolver` 出 Java 字段，`pipeline` 串成确定性解析流水线；`ws/` 解析工作存储为 WsNode 森林；`cobol_columns` 薄重导出相1 `preprocess.columns`。
- **analyzer/**（确定性分析，主线）：`callgraph` 调用链、`dataflow` 变量数据流，为翻译上下文供料，无 LLM。
- **translator/**（落地渲染，主线）：`rules` 规则引擎把 COBOL 语句确定性译为 Java（`segmenter` 先切句），`context/skeleton/naming` 建骨架与上下文，`postprocess/assemble` 后处理与组装；另含两条独立渲染产线 `wsaa/`（数据块）与 `skeleton_gen/`（主类骨架）。全部读 `config.spec_loader`。
- **validator/**（主线）：`java_validator` 校验 Java 产物风险（残留 TODO/GOTO/REDEFINES 计数）。
- **config/**（规范基座，步骤09 重构）：分三层——① 基础设施 `yaml_cache`（唯一 YAML 加载入口，按裸名解析子目录）/
  `conversions`（纯转换函数）/ `llm_config`（LLM 运行时配置）；② 规范正本 `specs/`（切分/骨架/WSAA 翻译规范，
  会持续新增层级）；③ 配套映射 `mappings/`（type/naming/copy/io）。访问层 `spec_loader`（翻译规范+IO+映射）与
  `grammar_loader`（切分+骨架文法）对外提供查询，**外部一律经访问层取规范、不直读 yaml**；config 不依赖任何业务模块。
- **parser.ws/**（WSAA 解析）：把 WORKING-STORAGE 块解析为 `WsNode` 森林（类型/宽度/88/REDEFINES 已回填）。
- **translator.wsaa/**（WSAA 渲染）：森林 → Java 类，叶子→字段、88→布尔方法、重叠组/REDEFINES→派生视图（步骤05 改为读 spec_loader）。
- **translator.skeleton_gen/**（骨架渲染，步骤05 骨架 + 步骤07 方法体）：CobolProgram → 主类骨架（主类 + 每 SECTION 方法带 wsaa 上下文参数 + COPY 依赖）；非 static、无单例、显式传参。步骤07 起 `body_context` 接 `segmenter`+`rules` 把方法体确定性译为 Java（`wsaa.` 前缀、`this.X(wsaa,…)` 调用），兜不住的叶子落 TODO，无 LLM。
- **根**：`main.py` CLI 驱动翻译线；`log_utils.py` 公共日志。
