## 1. 项目是什么

把单个大型 COBOL 程序（`ZPOLDWNM.cob`）翻译为 Java。核心是**本地确定性翻译**：

- **翻译主线（本地确定性，config 标准驱动，无 LLM）**：`config/*.yaml`（翻译标准正本，经
  `config/spec_loader` 访问）落地成本地翻译标准 → `parser/`（本地化解析：整程序 `cobol_parser`、
  工作存储 `ws/`）→ `translator/`（落地渲染：数据块 `wsaa/`、主类骨架 `skeleton_gen/`、语句级规则引擎
  `rules`）→ Java 产物（`assemble` 组装、`validator` 校验）。`analyzer/` 提供确定性调用链/数据流支撑。
- **AI 辅助层（`graph/`，LangGraph）**：编排上述确定性翻译，仅对规则兜不住的叶子语句调 LLM 兜底
  （+ Chroma RAG）。是开发期 AI 辅助，非生产主路径。
- **离线拆解线（确定性，无 LLM）**：`scripts/decompose.py` → `decompose/` 包，把整份 .cob 按规则切成
  独立小块，写文件 + 生成 `manifest.json`，可选幂等入库 MongoDB（`db/`）。供逐块翻译消费。

## 3. 调用关系（依赖方向，单向无环）

**拆解线**
- `scripts/decompose.py`（入口）→ `decompose.core`、`decompose.importer`
- `decompose.core` → `decompose.structure` / `decompose.blocks` / `decompose.manifest`
- `decompose.structure` → `decompose.lines`
- `decompose.blocks` → `decompose.lines` / `decompose.models`
- `decompose.manifest` → `decompose.blocks`(slug_filename)
- `decompose.importer` → `db.mongo_client.MongoStore`
- `scripts/verify_decompose.py`（独立校验，不被上述调用）→ 读 `<out>/manifest.json` 与 `<out>/拆解/*.cob`

**翻译主线（本地确定性，config 标准驱动）**
- `config.spec_loader` → `config/*.yaml`（标准正本）；下方各渲染器读它落地本地标准
- `parser.cobol_parser.parse` → CobolProgram；`parser.ws.parse_ws` → WsNode 森林；`parser.variable_resolver` → Java 字段
- `translator.wsaa` / `translator.skeleton_gen` / `translator.rules`(+`segmenter`) → Java（详见下方各产线）
- `analyzer.callgraph` / `analyzer.dataflow` → 确定性调用链/数据流（`dataflow` → `parser.variable_resolver`）

**AI 辅助层 graph/（LangGraph 编排 + 叶子兜底）**
- `main.py` → `graph.build_graph` → 4 节点：`parse_cobol`→`build_context`→`translate_section`(Send 并行×4)→`assemble`
- `parse_cobol_node` → `parser.pipeline.build_parse_result`（= `cobol_parser.parse` + `variable_resolver.resolve`）
- `build_context_and_skeleton_node` → `translator.context`（用 `analyzer.callgraph/dataflow` + `translator.naming/skeleton`）
- `translate_section_node` → `translator.segmenter` + `translator.rules`（确定性）；兜不住的叶子 → LLM + Chroma RAG → `translator.postprocess`
- `assemble_node` → `translator.assemble`（用 `translator.skeleton`）
- 各节点 → `graph.state.TranslationState`、`log_utils`

**WSAA 块翻译线（确定性，无 LLM；步骤03，步骤05 改造为规范驱动）**
- `scripts/translate_wsaa.py`（入口）→ `parser.ws.parse_ws` → `translator.wsaa.render_wsaa`
- `parser.ws`：`lines` →（`entry`/`conditions`/`value`/`pic`）→ `tree` → `model`
- `translator.wsaa`：`render_class` →（`render_field`/`render_condition`/`render_view`）；`render_view` → `storage`（定宽存储串）
- `render_field`/`render_class` → `config.spec_loader`（类型/初值/命名/样式串，步骤05 改造；产物回归不变）

**骨架翻译线（确定性，无 LLM；步骤05 骨架 + 步骤07 方法体）**
- `scripts/translate_skeleton.py`（入口）→ `parser.cobol_parser.parse` → `translator.skeleton_gen.render_skeleton`
- `translator.skeleton_gen`：`program_model`（CobolProgram→骨架要素）→ `copy_resolver`（COPY 角色分流）→ `render_skeleton`（装配主类）
- `skeleton_gen` → `config.spec_loader`（类名/COPY 角色/实体类）+ 复用 `translator.skeleton._section_to_method`（段名→方法名）
- 步骤07：`render_skeleton` → `body_context`（`build_body_ctx` 备料 → `segmenter`+`rules` 译方法体 → 瘦后处理）；
  方法体不再空体，规则兜不住的叶子落 `// TODO 叶子待译`（无 LLM）。复用 `parser.variable_resolver` + `translator.naming` + `translator.postprocess`
- 步骤13：paragraph 级 `PERFORM A THRU B` → `body_context._build_proc_order`（建四元过程单元顺序表注入 `Ctx`）；
  `rules._perform_range` 两级查找（SECTION 级零回归 / paragraph 级 `_perform_range_paragraph` 解析区间→登记 `Ctx.pending_range_methods`→返回 `this.aThruB();`，命名经 `spec_loader.perform_range_method`）；
  `render_skeleton` 收尾 drain pending → `body_context.render_pending_range_methods` 译合成区间方法 → 发射为类级方法
- 步骤14：THRU 区间内 GO TO 交互 → `_perform_range_paragraph` 登记改存**带标签**单元序列 `[(label, body),…]`（不丢标签）；
  drain 经 `body_context.translate_paragraphs_body`（跳过 `split_paragraphs` 有损往返，标签直传；`translate_section_body` 重构为其薄封装）→
  `rules.build_section(force_sm=True)` 重建区间内 paragraph 状态机：区间内 GO TO 回跳=循环 / 前向=精确跳转 / 出口=return / 区间外=保守 TODO-GOTO（`force_sm` 仅作用合成区间方法，普通 SECTION 渲染零变化）
- 步骤15（C-3）：单条 `PERFORM <paragraph>`（无 THRU）→ `rules._perform_single_paragraph`（`_perform_range` 无 THRU 分支改调）：
  SECTION→维持 `this.<sec>()`；paragraph（proc_order 恰一次、非 SECTION、合成名不撞 SECTION 方法）→登记单单元 pending `[(X, body)]`、
  调用点 `this.pXxx();` 不变（渲染期补出 `void pXxx(){…}`，B1 补实参）；兜不住→`this.pXxx()` + 可见 TODO（消除静默坏调用）。复用步骤13/14 渲染/命名机制

**规范访问层（步骤05；两条确定性产线共享）**
- `config.spec_loader` → `config/*.yaml`（type_mappings/naming_conventions/copy_mappings/wsaa_translation_spec）
  + 复用 `parser.variable_resolver._cobol_to_java_name`、`parser.ws.value.java_init`

## 5. 每个包 / 类的功能简述

- **decompose/**（拆解功能包，纯逻辑无 argparse）：`models` 定义 `Block`；`lines` 做列区清洗与停用/注释判定；
  `structure` 定位 DIVISION/COPY 并提取入参；`blocks` 切 SECTION 并构 WS/META 块；`manifest` 落盘切片与清单；
  `importer` 幂等入库；`core` 串起全流程。逐块职责单一、依赖单向。
- **db/**（数据访问层）：`MongoStore` 是 MongoDB 薄封装，pymongo 惰性导入，提供
  `ensure_unique_index/upsert(三态)/count/close`，幂等键 `(program, block_name)`，`$setOnInsert` 保护回填字段。
- **graph/**（AI 辅助编排层）：`state` 持翻译状态；`nodes` 4 节点编排确定性翻译、仅对兜不住的叶子调 LLM；`graph` 连成工作流（Send 并行派发 SECTION）。非生产主路径。
- **parser/**（本地化解析，主线）：`cobol_parser.parse` 解析整程序结构，`variable_resolver` 出 Java 字段，`pipeline` 串成确定性解析流水线；`ws/` 解析工作存储为 WsNode 森林；`cobol_columns` 是定列格式单一正本。
- **analyzer/**（确定性分析，主线）：`callgraph` 调用链、`dataflow` 变量数据流，为翻译上下文供料，无 LLM。
- **translator/**（落地渲染，主线）：`rules` 规则引擎把 COBOL 语句确定性译为 Java（`segmenter` 先切句），`context/skeleton/naming` 建骨架与上下文，`postprocess/assemble` 后处理与组装；另含两条独立渲染产线 `wsaa/`（数据块）与 `skeleton_gen/`（主类骨架）。全部读 `config.spec_loader`。
- **validator/**（主线）：`java_validator` 校验 Java 产物风险（残留 TODO/GOTO/REDEFINES 计数）。
- **config/**（规范基座，步骤09 重构）：分三层——① 基础设施 `yaml_cache`（唯一 YAML 加载入口，按裸名解析子目录）/
  `conversions`（纯转换函数）/ `llm_config`（LLM 运行时配置）；② 规范正本 `specs/`（切分/骨架/WSAA 翻译规范，
  会持续新增层级）；③ 配套映射 `mappings/`（type/naming/copy/io）。访问层 `spec_loader`（翻译规范+IO+映射）与
  `grammar_loader`（切分+骨架文法）对外提供查询，**外部一律经访问层取规范、不直读 yaml**；config 不依赖任何业务模块。
- **parser.ws/**（WSAA 解析）：把 WORKING-STORAGE 块解析为 `WsNode` 森林（类型/宽度/88/REDEFINES 已回填）。
- **translator.wsaa/**（WSAA 渲染）：森林 → Java 类，叶子→字段、88→布尔方法、重叠组/REDEFINES→派生视图（步骤05 改为读 spec_loader）。
- **translator.skeleton_gen/**（骨架渲染，步骤05 骨架 + 步骤07 方法体）：CobolProgram → 主类骨架（主类 + 每 SECTION 方法带 wsaa 上下文参数 + COPY 依赖）；非 static、无单例、显式传参。步骤07 起 `body_context` 接 `segmenter`+`rules` 把方法体确定性译为 Java（`wsaa.` 前缀、`this.X(wsaa,…)` 调用），兜不住的叶子落 TODO，无 LLM。
- **根**：`main.py` CLI 驱动翻译线；`log_utils.py` 公共日志。
