# 运行与验收

本文档记录当前仓库的最小可复现运行方式。项目分为两类路径：

- 确定性核心路径：解析、预处理、变量解析、ASG/visitor、规则翻译、骨架生成；不依赖 LLM。
- 全量翻译路径：LangGraph 编排、LLM 兜底、可选 RAG/Mongo；需要安装额外依赖和本地或远端 OpenAI 兼容模型服务。

## 1. 安装依赖

建议使用虚拟环境：

```bash
python -m venv .venv
.venv/Scripts/activate
python -m pip install -r requirements.txt
```

Windows PowerShell 可用：

```powershell
python -m venv .venv
.\.venv\Scripts\Activate.ps1
python -m pip install -r requirements.txt
```

## 2. 最小验收

不需要 LLM、不需要 MongoDB：

```bash
python scripts/check.py
```

默认等价：

```bash
python scripts/check.py --suite all
```

该命令会执行：

- Python 语法编译检查。
- 全量 unittest 回归。
- 使用 `tests/fixtures/minimal.cob` 跑 `main.py --parse-only`。
- 使用 `tests/fixtures/minimal.cob` 跑 deterministic skeleton 与 `main.py --no-llm` 输出检查。

当前基线：221 个测试通过，2 个本地 vLLM 测试在模型服务不可用时跳过。

并行开发 focused suite：

```bash
python scripts/check.py --suite quick
python scripts/check.py --suite leaf
python scripts/check.py --suite asg
python scripts/check.py --suite deterministic
python scripts/check.py --suite all
```

当前基线以 `scripts/check.py` 输出为准；本地 vLLM 测试在模型服务不可用时会跳过。

### 2.1 分组验收入口

`scripts/check.py` 支持按开发泳道缩小回归范围：

```bash
python scripts/check.py --suite quick
python scripts/check.py --suite leaf
python scripts/check.py --suite asg
python scripts/check.py --suite logic
python scripts/check.py --suite deterministic
python scripts/check.py --suite all
```

- `quick`：smoke、T16 deterministic report、最小 COBOL parse-only。
- `leaf`：统一 leaf 入口、STRING/UNSTRING/INSPECT/SEARCH、算术赋值、CALL、T13-T14 WS field view。
- `asg`：ASG/visitor、diff 工具主线样本、T10-T12 control-flow、T15 fallback observability。
- `logic`：逻辑文档生成器测试。
- `deterministic`：minimal skeleton 生成与 `main.py --no-llm` 输出。
- `all`：默认完整本地主线闸口，包含全量 unittest、最小 COBOL parse-only、骨架生成、no-LLM translation。

T15 后，`--suite asg` 已计入 `TestMainlineSectionViaAsg`，覆盖 ASG 异常回退 legacy 时的结构化 fallback 事件统计。

T16 后，`quick` 已计入 `test_t16_deterministic_report`，覆盖多输入确定性报告 JSON、输出文件列表、TODO/风险计数和 ASG fallback summary。

翻译规范正本还包括 `docs/翻译标准/确定性验收.md`；ASG fallback 统计与 deterministic report 的字段要求以该文件为准。

### 2.2 ASG vs legacy diff 闸口

`scripts/diff_asg_vs_legacy.py` 用于逐字符比对旧路与 ASG/visitor 新路。当前 `VERB_MATRIX` 覆盖：

```text
MOVE IF PERFORM PERFORMCALL FLOW IO CALL ARITH FALLBACK CONTROL SECTION
```

常用示例：

```bash
python scripts/diff_asg_vs_legacy.py tests/fixtures/minimal.cob --verb SECTION
python scripts/diff_asg_vs_legacy.py tests/fixtures/minimal.cob --verb FALLBACK
```

## 5. 生成确定性骨架

不走 LLM 的整程序翻译主入口：

```bash
python main.py tests/fixtures/minimal.cob --no-llm --output output
```

这会输出单个 Java 文件，例如 `output/Minismoke.java`。该路径不导入 LangGraph，不调用 vLLM；规则覆盖不了的复杂语句会保守输出 TODO。

底层骨架脚本也可以直接使用：

```bash
python scripts/translate_skeleton.py --in tests/fixtures/minimal.cob --out output/Minismoke.java
```

### 5.1 大样本翻译冒烟

对大型 COBOL 程序（数百 SECTION）做快速冒烟验证：

```bash
# 仅翻译前 10 个 SECTION（快速冒烟）
python scripts/translate_skeleton.py --in large_program.cob --out output/Large.java --limit 10

# 全量翻译 + 详细统计
python scripts/translate_skeleton.py --in large_program.cob --out output/Large.java --stats

# 通过 --no-llm 路径全量翻译
python main.py large_program.cob --no-llm --output output/large
```

`--limit N` 参数限定翻译前 N 个 SECTION，适合开发期快速验证骨架生成不崩溃。`--stats` 参数输出 TODO 计数、叶子 TODO、LLM 调用数等翻译质量指标。

## 6. 全量翻译

全量翻译依赖 `langgraph` 和 LLM 相关依赖。默认模型端点是：

```text
http://localhost:8000/v1
```

可通过环境变量切换：

```bash
LLM_BASE_URL=http://localhost:8000/v1
LLM_API_KEY=dummy
LLM_MODEL=qwen3-8b-GPTQ
```

运行：

```bash
python main.py your_program.cob --output output
```

快速限定前 N 个 SECTION：

```bash
python main.py your_program.cob --sections 5 --output output
```
