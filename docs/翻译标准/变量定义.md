## 一、程序整体结构与翻译约定

### 1）PROGRAM-ID

```cobol
PROGRAM-ID.     ZPOLDWNM
```

`ZPOLDWNM` 为**新建类名**。

### 2）WORKING-STORAGE SECTION

- 参数定义块。
- 全局参数名字 → 改成 **cob 名字 + wsaa**，即 `ZPOLDWNMWSAA`。

### 3）COPY 部分

```cobol
COPY ITEMKEY.
COPY GETDESCREC
COPY AGLFSKM
COPY VARCOM.
```

`COPY` 表示**引用其他 cob 文件**，被引用的 cob 各自职责不同：

| COPY 结尾 | 含义 |
|---|---|
| `REC` / `SKM` 结尾 | 基本是**实体类的字段定义** |
| `VARCOM` | 可以是**处理逻辑**，当前 cob 会调用它的方法 |

### 4）入参

```cobol
PROCEDURE DIVISION           USING LETCMNT-PARAMS
                                   PMSPNT-PARAMS.
```

表示**方法入口、入参**。

### 5）SECTION

`SECTION` 表示**方法体**，每个 `SECTION.` 截取出一个文件。

## 三、WORKING-STORAGE SECTION 翻译规范

```cobol
WORKING-STORAGE SECTION.
   01  WSAA-PROG                   PIC X(07) VALUE 'ZPOLDWN'.       <CSV001>
   01  WSAA-VERSION                PIC X(02) VALUE '01'.            <CSV001>
   01  WSAA-LETOKEYS.                                              <CSV001>
       03  WSAA-LANGUAGE           PIC X(01).                      <CSV001>
       03  WSAA-TSALEUNT           PIC X(08).                      <CSV001>
       03  WSAA-B-CHDRNUM          PIC X(08).                      <A41758>
       03  FILLER                  PIC X(13).                      <A41758>
  *    03  FILLER                  PIC X(21).              <A41758><CSV001>
   01  WSAA-STATUZ                 PIC X(04).                      <CSV001
```

- `WORKING-STORAGE SECTION` 这部分是**全局变量**，所有方法都可以读到。
- 翻译命名：**cob 名字 + wsaa**。

### 基础标量

```cobol
01  WSAA-PROG                   PIC X(07) VALUE 'ZPOLDWN'.
01  WSAA-CSV-RIDER-SEQ          PIC 9(03).
```

- `X(07)`：字符串，`WSAA-PROG` 默认值：`ZPOLDWN`。

### 组合字段与截取（group / FILLER）

- `WSAA-LETOKEYS` 长度 = 1 + 8 + 8 + 13。
- `WSAA-TSALEUNT`：是 `WSAA-LETOKEYS` 的截取，**第 9 位 + 8 长度**。
  - 修改 `WSAA-TSALEUNT` 会相应改 `WSAA-LETOKEYS`；
  - 反之，`WSAA-LETOKEYS` 的第 9 到 17 位修改，会改动 `WSAA-TSALEUNT`。
- `03 FILLER` 相当于**占位符**。

### 88 条件（布尔判断）

```cobol
01  WSAA-A65086.                                                  <A64650>
    03 WSAA-VAL28-PRD           PIC X(04).                        <A64650>
    88 IS-6PWB                  VALUES '6PWB'.                    <A64650>
    88 IS-WPDH                  VALUES 'WPDH'.                    <A64650>
    88 IS-2DWE                  VALUES '2DWE'.                    <A64650>
    88 IS-DEEJ                  VALUES 'DEEJ'.                    <A64650>
    03 WSAA-6PWB-FLG            PIC X(01).                        <A64650>
    03 WSAA-WPDH-FLG            PIC X(01).                        <A64650>
    03 WSAA-2DWE-FLG            PIC X(01).                        <A64650>
    03 WSAA-DEEJ-FLG            PIC X(01).                        <A64650
```

- `WSAA-A65086` 长度是 `X(04) + X(01) + X(01) + X(01) = 8`。
- `88 IS-6PWB` 表示**判断条件**：
  - 代码里用法：`IF IS-6PWB THEN MOVE 'Y'`，表示 `WSAA-VAL28-PRD` 值 = `6PWB` 时 `IS-6PWB` 为 true；
  - 在赋值 `WSAA-VAL28-PRD` 处判断。

### 数值类型（PIC 9 / S9 / V / COMP-3）

```cobol
XX  PIC 999 VALUE 0
```

表示该变量可存储 3 位数字（每位 0-9），无符号，默认值为 0。

```cobol
01  WSAA-PREM-TOTAL             PIC S9(15)V9(2)
01  WS003-PREM-UNIT             PIC S9(03)  COMP-3
01  WSAA-ORIGAMT                PIC 9(15)V99
```

**`WSAA-PREM-TOTAL` 解析：**

- `S`：可存储正号或负号；
- `9(15)`：整数部分最多 15 位；
- `V`：隐式小数点，实际不占存储空间，只是定位点；
- `9(2)`：2 位数字，小数部分 2 位（分/角）；
- 总位数：15 + 2 = 17 位数字。
- 取值范围：
  - 最大值：`+999,999,999,999,999.99`（约 1 千万亿）
  - 最小值：`-999,999,999,999,999.99`
- `V99`：表示 2 位小数。

**`WSAA-ORIGAMT` 解析：**

- 取值范围：`0` 到 `999,999,999,999,999.99`。

**`S9(03) COMP-3` 解析：**

- `COMP-3`（压缩十进制）是一种 **每字节存 2 个数字 + 最后半个字节存符号** 的存储方式。
- 存储计算：`(数字位数 + 1) / 2`，向上取整。
  - 本例存储 `(3 + 1) / 2 = 2` 字节：
    - 字节 1：第 1-2 位数字
    - 字节 2：第 3 位数字 + 符号（半字节）
- 符号存储：十六进制 `C`（正）、`D`（负）、`F`（无符号）。

值 `123` 的 COMP-3 存储（十六进制）：

```
┌────────────┬────────────┐
│   字节1     │   字节2     │
│ 12 (X'12') │ 3C (X'3C') │
└────────────┴────────────┘
X'3C' 表示：数字 3 + 正号(C)
```

值 `-456` 的 COMP-3 存储：

```
┌────────────┬────────────┐
│   字节1     │   字节2     │
│ 45 (X'45') │ 6D (X'6D') │
└────────────┴────────────┘
X'6D' 表示：数字 6 + 负号(D)
```

### 默认值 SPACES

```cobol
01  WSAA-SKIP-COMP              PIC X(01) VALUE SPACES.
```

默认值为 `""`。

### OCCURS（集合 / 数组）

```cobol
01  WS001-CVALUES .
    03  WS001-CVALUE                   PIC S9(12)  COMP-3
                                       OCCURS 200.
```

代码里使用：`MOVE CRAT-CYRETENT01 TO WS001-CVALUE(100)`，`WS001-CVALUE` 类似**集合**。

### 二维数组 + REDEFINES（重定义）

```cobol
01  WSAA-CPB-INF.
    03  WSAA-CPB-LINE OCCURS 27 TIMES.
       05 FILLER                PIC X(3)  VALUE 'CP'.
       05 WSAA-CPB-R.
          07 WSAA-CPB-ROW OCCURS 4 TIMES.
             09 WSAA-CP-AGETAG         PIC X     VALUE '#'.
             09 WSAA-CP-AGE            PIC X(03).
             09 WSAA-CP-AMTTAG         PIC X     VALUE '#'.
             09 WSAA-CP-AMOUNT         PIC X(18).
       05 WSAA-CP-ROW-STR REDEFINES WSAA-CPB-R.
                07 FILLER             PIC X(92).
```

使用：

```cobol
MOVE    CSV-AMT-TEMP(1:15) TO
                   WSAA-CP-AMOUNT(CSV-L , CSV-R)(1:15).
MOVE    CSV-AMT-TEMP(16:2) TO
                   WSAA-CP-AMOUNT(CSV-L , CSV-R)(17:2).
MOVE    '.'    TO   WSAA-CP-AMOUNT(CSV-L , CSV-R)(16:1).
MOVE    '#'    TO   WSAA-CP-AGETAG(CSV-L , CSV-R).
MOVE    '#'    TO   WSAA-CP-AMTTAG(CSV-L , CSV-R).
```

**说明：**

- `WSAA-CP-AMOUNT` 是**二维数组**，`CSV-L` 最大为 27，`CSV-R` 为 4；
- `WSAA-CP-ROW-STR` 为**重定义**：`REDEFINES WSAA-CPB-R`，长度 `(1+3+1+18)*4 = 92`。
- `REDEFINES`：相当于重新定义，上面相当于将**集合转化为字符串**。

### 数组上下文内 REDEFINES 视图规范

本项目全局约定：`REDEFINES` 位于 `OCCURS` 祖先链下时，不得生成无下标的标量 getter/setter，也不得把整个 Java 数组当作单个定宽存储串切片。可证明为同一数组元素内共享存储的形态，必须生成带下标参数的派生视图。

允许转正的形态：

- `REDEFINES` 目标和 alias 处于同一组 `OCCURS` 祖先维度下。
- 目标可表示为定宽 DISPLAY 存储：字符/数值标量，或子项均为字符/数值标量的组。
- alias 子项为字符/数值标量，或字符数组子项。
- 生成方法必须携带祖先维度下标；一维为 `getX(int i)` / `setX(int i, ...)`，二维为 `getX(int i, int j)` / `setX(int i, int j, ...)`。
- 方法内部用 COBOL 1-based 下标访问 Java 数组元素：`field[i - 1]`、`field[i - 1][j - 1]`。

仍需保守 TODO 的形态：

- 目标和 alias 的祖先 `OCCURS` 维度不一致。
- 目标含 `COMP-3`、编辑型 PIC、无法定宽表示的嵌套组。
- alias 含无法明确 offset 的嵌套组，或数值数组子项。
- 无法证明每个 alias 元素只覆盖同一个 backing 元素。

## 四、cob 参数定义整体说明

- `WSAA-CPB-INF` 为 `01`，包含 `05` 分别为：`FILLER`、`WSAA-CPB-R`、`WSAA-CP-ROW-STR`。
- `WSAA-CPB-R` 为 `05`，包含 `07`，为 `WSAA-CPB-ROW OCCURS 4 TIMES`，一个长度 4 的集合。
  - `07` 不是基础类型，像是我们自定义的类，里面有 4 个字段：`WSAA-CP-AGETAG`、`WSAA-CP-AGE`、`WSAA-CP-AMTTAG`、`WSAA-CP-AMOUNT`。

## 补充规范：OF / IN 字段限定

COBOL 允许用 `A OF B` / `A IN B` 表示“父组 B 下的字段 A”。本项目全局约定：字段限定必须显式消歧，不得把 `A OF B` 静默当作普通裸词 `A`，也不得在存在歧义时任选一个字段。

允许转正的形态：

- 限定引用能解析为唯一字段，例如 `STATUS OF HEADER`。
- 多级限定按从字段到父组的顺序记录，例如 `AMOUNT IN DETAIL IN ORDER-REC`。
- 解析必须通过显式限定名映射或等价的 WS 树唯一索引。

仍需保守 TODO 的形态：

- 限定链无法解析。
- 同一限定后缀命中多个字段。
- 语句层还没有把多 token `A OF B` 组装成单个限定引用。

未解析限定字段在生成表达式时必须保留可见 TODO，例如：

```java
"/* TODO unresolved qualified field: STATUS OF HEADER */"
```
