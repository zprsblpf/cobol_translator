# 确定性验收与报告规范

本文定义本项目全局有效的确定性翻译验收规则。它与 `变量定义.md`、`流程结构.md`、`io查询.md` 并列，约束翻译器输出、fallback 可观测性和机器可读报告。

## 验收入口

主线准入以 `python scripts/check.py` 为准；分组回归仅用于按开发范围缩小本地验证面，不能替代完整主线闸口。

当前分组职责：

- T16 deterministic report 属于 `quick`。
- T13-T14 WS field view 属于 `leaf`。
- T10-T12 control-flow 属于 `asg`。
- T15 fallback observability 属于 `asg`。

## ASG fallback 可观测

主线 SECTION 渲染默认走 ASG / visitor 路径。ASG 渲染异常时允许保守回退 legacy renderer，但必须记录结构化事件，不能静默回退。

每个 fallback 事件至少包含：

- `exception_type`
- `message`
- `paragraph_labels`
- `force_sm`

验收要求：

- 可通过 `asg_fallback_summary(ctx)` 获取 `{count, events}`。
- `scripts/check.py --suite asg` 必须覆盖 ASG fallback 统计。
- 预期不 fallback 的样本出现 fallback 时，测试应能失败或显式暴露计数。

## deterministic 多文件报告

确定性报告必须是机器可读、稳定排序、可重复写出的 JSON。

报告顶层字段：

- `schema_version`
- `command.inputs`
- `command.output_dir`
- `command.report_path`
- `files[]`
- `totals`

每个 `files[]` 项至少包含：

- `input`
- `output`
- `program_id`
- `class_name`
- `line_count`
- `todo_count`
- `risk_count`
- `risks`
- `asg_fallback_summary`

稳定性要求：

- 输入路径按稳定顺序处理。
- JSON 使用固定缩进和 key 排序。
- TODO、风险、ASG fallback 计数必须可汇总到 `totals`。
- 报告命令不得依赖 LLM、MongoDB 或外部服务。

验收入口：

```powershell
python -m unittest test_t16_deterministic_report -v
python scripts/check.py --suite quick
```
