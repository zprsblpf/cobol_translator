## 一、GO TO 跳转

### 1）本 shop 方言：`GO` 可省略 `TO`

本 shop 的源码里，部分跳转省略了标准 COBOL 的 `TO`，直接写作 `GO <段名>`：

```cobol
GO 2060-RECEIPT
GO 3190-EXIT
GO 5320-CALL
```

实测（2026-06-25）：
- `ZPOLDWNM.cob`：**13 处**省 TO，全部为真跳转（目标均为真实 paragraph 名，且带 `<CSVxxx>`/`<Axxxxx>`
  变更标记），无字符串/注释假阳；
- `SBSURCALC.cbl`：1 处（`GO C290-EXIT`）；
- 其余样例：0 处。

标准 COBOL 与多数工具只认 `GO TO`，故这些省写的跳转若不处理会被**漏检**（控制流缺失、翻译出错）。

### 2）标准：解析前归一为 `GO TO`

**规范**：在预处理阶段（相1）把 `GO <段名>` 归一为标准 `GO TO <段名>`，使后续控制流识别/翻译一致。
归一是**源码层文本归一**，不改变语义，只补回省略的 `TO`。

**归一规则（供 config 机器模板派生）**：
- 命中：单词边界的 `GO` + 空白 + 其后**不是** `TO` + 紧跟段名首字符（字母/数字）。
- 动作：补成 `GO TO `（即在 `GO` 与段名间插入 `TO `）。
- **边界（必须遵守）**：
  - 不动 `GOBACK`（`GO` 后无空白，不命中）；
  - 不动已是 `GO TO` 的（幂等）；
  - **引号内文本不动**（如 `MOVE 'GO HOME' TO X` 里的 `'GO HOME'` 不归一）——引号外才是语句。

### 3）GO TO → Java 翻译

GO TO 归一后，其到 Java 的降级翻译沿用既有控制流策略（`config/specs/skeleton_spec.yaml` 的 `control_flow`
/ back-edge 状态机），本标准不改动，仅保证「省 TO 的跳转先被归一、不漏检」。
