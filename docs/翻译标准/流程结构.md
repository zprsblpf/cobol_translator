# 流程结构（控制流）— COBOL → Java 翻译标准

> 本文件定义流程结构 / 控制流类语句的标准。
> 与 `变量定义.md`（变量类）、`io查询.md`（IO 查询类）并列。

## 一、GO TO 跳转

详见 `流程结构-GOTO翻译规范.md`。

1. 方言归一：`GO <段名>` → `GO TO <段名>`
2. 翻译方案：
   - **出口跳转**（GO TO xxx-EXIT）→ `return;`
   - **已知 SECTION** → `this.method(); return;`
   - **段内 paragraph 跳转** → while+switch 状态机
   - **未知目标** → `// TODO-GOTO` 人工核对
3. GO TO DEPENDING ON → TODO-GOTO-DEPENDING 占位

## 二、IF / ELSE

### 1）标准形态

```cobol
IF WSAA-STATUZ = O-K
   PERFORM 1000-PROCESS
END-IF
```
```java
if ("O-K".equals(wsaaStatuz)) {
    process1000();
}
```

### 2）OR 值列表简写

`IF A = "X" OR "Y"` → `if ("X".equals(a) || "Y".equals(a))`

当条件分段缺少关系运算符时，复用前一段的左操作数和操作符。

### 3）复杂条件 fallback

- 88 条件名、NOT NUMERIC、参考修改（`A(IX:1)`）、多层括号嵌套 → TODO-IF

## 三、EVALUATE / WHEN

```cobol
EVALUATE WSAA-CODE
    WHEN 'A' PERFORM 100-A
    WHEN OTHER CONTINUE
END-EVALUATE
```
```java
switch (wsaaCode) {
    case "A": p100A(); break;
    default: break;
}
```

## 四、PERFORM

| COBOL | Java |
|-------|------|
| `PERFORM SECTION` | `this.method(wsaa, args);` |
| `PERFORM paragraph` | 提取为独立方法 |
| `PERFORM A THRU B` | 展开为连续调用或合成区间方法 |
| `PERFORM UNTIL cond` | `while (!(cond))` |
| `PERFORM VARYING i` | `for (i...; !(ci); i += step)` |
| `n TIMES` | `for (int _i = 0; _i < n; _i++)` |

## 五、SEARCH

### 1）首版保守子集：显式 VARYING 的线性搜索

`SEARCH` 首版只固化简单线性扫描：

```cobol
SEARCH WSAA-ROWS VARYING WSAA-I
    WHEN WSAA-ROW-CODE(WSAA-I) = WSAA-KEY
```

翻译为 Java `for` 循环，索引按 COBOL 1-based 语义从 1 扫到数组长度；`WHEN` 条件命中时 `break`，数组下标形式仍沿用现有叶子表达式与后处理规则。

```java
for (wsaaI = 1; wsaaI <= wsaaRows.length; wsaaI++) {
    if (wsaaKey.equals(wsaaRowCode(wsaaI))) {
        break;
    }
}
```

### 2）不支持范围必须结构化可观测

以下形态不臆造翻译，输出 `UNSUPPORTED[LEAF.SEARCH.001]`：

- `SEARCH ALL`；
- `AT END` / `NOT AT END` 分支；
- 缺少显式 `VARYING` 索引；
- 多个 `WHEN` 或带 inline action body 的复杂 SEARCH；
- SEARCH 目标不是已知数组，或 `WHEN` 条件无法由确定性条件翻译接住。

## 六、INSPECT 叶子语句

`INSPECT` 作为叶子语句走统一 leaf 入口；当前只固化无作用域限定的保守子集。

确定性支持：
- `INSPECT <source> TALLYING <counter> FOR CHARACTERS` → `<counter> += String.valueOf(<source>).length();`
- `INSPECT <source> TALLYING <counter> FOR ALL <needle>` → 按非重叠出现次数累加到 `<counter>`。
- `INSPECT <source> REPLACING ALL <needle> BY <replacement>` → `String.valueOf(<source>).replace(...)` 后写回源字段。
- `INSPECT <source> REPLACING FIRST <needle> BY <replacement>` → `replaceFirst(Pattern.quote(...), Matcher.quoteReplacement(...))` 后写回源字段。
- `INSPECT <source> REPLACING LEADING <needle> BY <replacement>` → 只替换开头连续命中的 `<needle>` 后写回源字段。

暂不支持：
- `BEFORE` / `AFTER` 限定范围。
- `CONVERTING`。
- 单条 `INSPECT` 内多个 `TALLYING` 或 `REPLACING` 子句。

不支持形态必须返回未命中，由上层生成结构化 unsupported 标记，不静默降级。

## 五、UNSTRING 叶子语句

`UNSTRING` 作为叶子语句走统一 leaf 入口；当前只固化单源、单分隔符、顺序写入目标字段的保守子集。

确定性支持：
- `UNSTRING <source> DELIMITED BY SPACE INTO <target>...`
- `UNSTRING <source> DELIMITED BY <literal-or-field> INTO <target>...`

翻译时以 `String.valueOf(<source>).split(Pattern.quote(<delimiter>), <limit>)` 拆分，按目标字段顺序写入；目标数量为 1 时只取首段，目标数量大于 1 时最后一个目标保留剩余文本。生成代码用局部代码块包住 `__unstringParts` 临时数组，避免同一 Java 方法内多条 `UNSTRING` 重复声明冲突。

暂不支持且必须结构化可观测：
- `WITH POINTER` → `UNSUPPORTED[LEAF.UNSTRING.POINTER.001]`
- `TALLYING IN` / `COUNT IN` → `UNSUPPORTED[LEAF.UNSTRING.COUNT.001]`
- `DELIMITER IN` → `UNSUPPORTED[LEAF.UNSTRING.DELIMITER.001]`
- `ON OVERFLOW` / `NOT ON OVERFLOW` → `UNSUPPORTED[LEAF.UNSTRING.OVERFLOW.001]`
- `ALL` 或 `OR` 组合分隔符 → `UNSUPPORTED[LEAF.UNSTRING.DELIMITER.002]`

缺少 `DELIMITED BY`、缺少 `INTO`、`DELIMITED BY SIZE` 等非目标子集形态返回未命中，由上层统一 unsupported 机制兜底。
