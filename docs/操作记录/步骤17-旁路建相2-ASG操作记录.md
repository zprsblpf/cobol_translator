## 3. 验收四项 → 单测落实

- **① 结构提升**（`TestAsgBuild.test_build_runs_and_sections_match_parse` / `test_registry_attached_to_root`）：
  内联两 SECTION + EXIT paragraph 程序 `parse`→`build_asg`，`program_id`/SECTION 数与 parse 一致、
  节点树非空、`Program.registry` 是 `ProcRegistry` 且含 `1000-MAIN`/`1000-EXIT` 单元。
- **② 引用解析**（`test_goto_target_resolved_to_unit`）：`GO TO 1000-EXIT` 提升出 `GotoStmt`，
  `target.name=="1000-EXIT"` 且 `target.unit` 解析到真实 paragraph 单元（非 None）。
- **③ THRU 区间**（`TestAsgRegistryThru` 5 用例）：`resolve_thru("A","B")` 取 `[A,M,B]` 闭区间
  （中间不丢、Z 不越界）；B 缺失/`NOPE`→`[A]`；B 在 A 前→`[A]`；A 未知→`[]`；按名大小写不敏感。
- **④ 单点自证**（`TestAsgGotoVisitor` 3 用例）：`GotoJavaVisitor.visit(GotoStmt(…))` 对 `…EXIT`
  与未知段的输出 == `rules._sk_control`（`flow_label=None`/`indent=0`/目标不在 `known_sections`）逐字符；
  目标缺失→`["return;"]`。
