# 步骤37-逻辑文档生成器操作记录

## 目标

新增 `logician/` 模块，从 ASG 分析 COBOL 程序 → 结果表反向追溯全路径 → JSON + Markdown 双输出。

## 本次修改

### 新建文件

| 文件 | 说明 |
|------|------|
| `logician/__init__.py` | 门面：generate / save / generate_and_save |
| `logician/models.py` | 数据模型：LogicDoc, LogicNode, LogicEdge, LogicPath, ResultLeaf, IOCall, EdgeCondition |
| `logician/tokens.py` | 词元提取：extract_tokens / extract_io_call / expand_88_condition |
| `logician/results.py` | 结果叶子识别：find_results 识别 5 类结果叶子 |
| `logician/builder.py` | 核心构建器：ASG 解析 → 调用链 → DFS 路径追踪 → 交叉点合并 → LogicDoc |
| `logician/output/__init__.py` | 输出包标记 |
| `logician/output/json_output.py` | LogicDoc → JSON 序列化（稳定 key 排序） |
| `logician/output/md_output.py` | LogicDoc → Markdown 逻辑树文档渲染 |
| `scripts/generate_logic_doc.py` | CLI 入口：`--cob <file> --out-dir <dir>` |
| `tests/test_logician.py` | 45 个单元测试 |

### 设计要点

1. **复用已有分析器**：ASG builder (`asg.builder.build`) + analyzer.callgraph 提供段级调用链
2. **词元提取独立**：每 ASG 节点类型有独立 _tokenize_<Type> 分派，覆盖 MOVE/CALL/IF/EVALUATE/PERFORM/GO TO 等
3. **段级 DFS 路径追踪**：从入口段沿 PERFORM 调用链追踪到结果叶子，记录节点序列和条件摘要
4. **交叉点合并**：被多条路径共享的节点标记 path_tags
5. **双层输出**：JSON（机器可读）+ Markdown（人工审阅，含流程图）

### 词元类型覆盖

| 类别 | 词元 | 触发条件 |
|------|------|---------|
| 赋值 | ASSIGN / RESET / FUNC_SET | MOVE 的目标语义 |
| 条件分支 | BRANCH_IF / BRANCH_SWITCH | IF / EVALUATE |
| 调用 | CALL_PROC / CALL_PROG / LOOP | PERFORM / CALL |
| 跳转/返回 | JUMP / RETURN | GO TO / EXIT/GOBACK |
| DB IO | DB_READR / DB_BEGN / DB_WRITR / DB_UPDAT / DB_DELET | IoWrite/IoRead/Begn 节点 |
| 其他 | MATH / TEXT_OP / SEARCH / FATAL / UNRESOLVED | 动词首词匹配 |
