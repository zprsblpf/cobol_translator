## 一、产物路径（本项目）

1. **拆解/翻译产物** → `/home/zp/Documents/cob/拆解/{PROGRAM}/`（按用户指定）。
   （设计文档 `docs/详细设计/`、操作记录 `docs/操作记录/`、脚本 `scripts/` 等通用约定见全局规范。）

## 二、COBOL 领域固定约定（已确认）

2. **注释/停用判定**：
   - 指示列（第 7 列）= `*` 或 `/` → **注释**，**保留**（含业务语义，对翻译有用）。
   - 第 1–6 列 = `!!!!!!` → **停用旧代码**，**丢弃**（含 `!!!!!!` 开头的 SECTION 头，不作为边界）。
3. **拆解命名规则**：
   - `PROGRAM-ID. xxx` → 类名 `xxx`。
   - `WORKING-STORAGE SECTION` → 参数定义块，命名 `{PROGRAM}WSAA`。
   - `COPY`：`REC`/`SKM` 结尾 ≈ 实体类字段定义；含 `VARCOM` ≈ 处理逻辑/方法；其余 = other（只分类，不展开）。
   - `PROCEDURE DIVISION USING ...` → 方法入口 + 入参。
   - PROCEDURE 内每个有效 `SECTION` → 一个方法体 / 一个切片文件。
4. **切片保真度**：去掉行号列（1–6）与行尾 `<变更标记>`，保留第 7–72 列代码区与 `*` 注释。

## 三、知识库（MongoDB）

5. 库 `cobol`，集合名 = cob 程序名（每程序一表），每个块一条文档。
6. 文档预留字段 `params_in / params_out / calls / logic / java_code`，按步骤逐步回填。
7. 写库用**幂等 upsert**（键 `{program, block_name}`），重跑不重复、不覆盖已回填字段。
8. Mongo 不可用时**先写文件**，`manifest.json` 含全部入库字段，就绪后用同脚本 `--import-only` 补入库。

## 四、六步流水线（总览）

1. **拆解 + 存库**（不翻译）← 已完成
2. 每块的处理逻辑 + 代码生成规则
3. 按块生成 Java 代码
4. 组装
5. 校验
6. 按原 cob 校验逻辑

> 约定：Java 生成后库内方法体可能变更，故库结构支持更新（预留字段 + upsert）。
