## 1. 本步目标与非目标

**目标**：建立 `preprocess/` 包，对外暴露**两件产物**——① 方言列处理原语（停用/注释/调试判定 + 列区清洗，
即现 `cobol_columns` 全套）；② **「干净行流」`CleanSource`**（带原始行号回溯），把现在散在
`cobol_parser` 里、每个消费点各自即时重清洗的 `_strip_cobol_line`/`_clean_lines` 收口为相1 的**唯一产物**。

**非目标（本步明确不做，已排期到后续步骤）**：
- 不建 ASG、不动 segmenter 内部切分逻辑（相2 的事）。
- 本步**不并入方言归一**，但它确属相1（初步设计 §1「COPY 展开·格式归一」），故**已排期**而非搁置：
  - **步骤16「相1 方言归一吸收」**：并入 `CRLF 归一`（幂等无副作用，零行为变化）+ `GO→GO TO 归一`
    （主线现 `_GOTO_RE` 只认 `GO TO`，省 `TO` 的 `GO 段名` 被漏检；归一后新检出 → `go_tos`/`review_items`
    增多，是**行为变更**，按初步设计 §5.3 走「前后产物比对 + 人工确认更正确」，故必须与本步零变化第一刀分开）。
  - **COPY 展开**（真·展开，非 spike 桩生成）：单独成步骤。spike `gen_stub_copybooks` 是 ProLeap 缺簿拐杖，
    我们已决定不接 ProLeap（见 `proleap-parser-evaluation`），主线缺簿不报错（仅收 `copy_refs` 列表），
    故桩生成**不并入**；真·COPY 展开（解析拷贝簿正文注入行流）是实打实新特性，独立设计/认可。
  - spike 脚本本步**原样保留**在 `scripts/spike_proleap/`。
- 不改 DIVISION/SECTION 定位算法本身（仅让它改从相1 取原语，逻辑字符不变）。

## 2. 相1 包结构与各文件职责

```
preprocess/                     ← 新建「相1」包
├── __init__.py                 瘦门面：导出原语 + CleanSource + preprocess_file()
├── columns.py                  ← parser/cobol_columns.py 迁入（方言列处理正本，逻辑零改动）
└── line_stream.py             「干净行流」产物：CleanSource + 由物理行流构建
```

调用关系（数据流）：

```
源 .cob ──readlines──► preprocess.preprocess_file()
                          │  ├─ columns.*（逐行判定/清洗原语）
                          │  └─ line_stream 组装 ──► CleanSource（相1 产物）
                          ▼
        旧 cobol_parser.parse() / segmenter 消费 CleanSource（接线见 §4）
```

### 2.1 `preprocess/columns.py`（方言列处理正本）
- **做什么**：原 `parser/cobol_columns.py` **整文件迁入**，函数与逻辑**逐字不改**：
  `is_deactivated` / `is_comment` / `is_debug` / `indicator` / `clean_line` / `effective` / `clean_block`。
- **设计思路**：列处理是相1 的最底层原语，本就是单一正本（步骤06），迁入后正本归属相1，名实相符。
  对 `grammar_loader.column_model()` 的延迟导入保持原样（避免加载期环依赖）。

### 2.2 `preprocess/line_stream.py`（干净行流产物）
- **类 `CleanLine`**（dataclass）：一条有效代码行的回溯单元。
  字段：`lineno: int`（原始物理行号，1-based）、`code: str`（净化后代码区文本）。
  *做什么*：把「行号→代码」绑定成对象，替代现在裸 `tuple[int, str]`，让相2/下游能稳定回溯原始行。
- **类 `CleanSource`**（dataclass）：相1 对一份源文件的完整产物。
  字段：
  - `raw_lines: list[str]`——原始物理行（保留，供仍需原始索引的旧逻辑回溯，零行为变化的关键）；
  - `clean_lines: list[CleanLine]`——**去注释/停用/空行后的有效行流**（即旧 `_clean_lines` 产物）；
  - `total_lines: int`——物理总行数。
  方法：
  - `strip_line(raw: str) -> str`：净化一物理行（停用/注释/调试→`""`，否则取代码区）。
    *实现*＝原 `cobol_parser._strip_cobol_line` 逐字搬入，调用 `columns.*`，逻辑不变。
  - `as_pairs() -> list[tuple[int, str]]`：把 `clean_lines` 摊回 `(行号, 代码)` 元组列表，
    **为旧调用方提供零改动适配**（旧 `_clean_lines()` 的返回形态）。
- **函数 `build(raw_lines: list[str]) -> CleanSource`**：遍历物理行，对每行 `strip_line`，
  非空者收为 `CleanLine`，组装 `CleanSource`。*实现*＝原 `cobol_parser._clean_lines` 等价逻辑，
  仅产物从裸 tuple 升级为对象。

### 2.3 `preprocess/__init__.py`（瘦门面）
- 导出列原语（`is_deactivated` 等全套）+ `CleanLine` / `CleanSource`。
- **函数 `preprocess_file(path) -> CleanSource`**：`open(...).readlines()` → `line_stream.build()`。
  这是相1 对外**唯一入口**，封装「读文件 + 构建干净行流」。*只做编排，不放逻辑*（瘦入口规范）。

## 4. 绞杀接线（保证「零行为变化」，旧路径不破）

按 decompose/lines.py 已验证的**薄重导出**手法，让旧 import 路径全部仍可用：

1. **`parser/cobol_columns.py`** → 改为从 `preprocess.columns` **薄重导出**全套 API（保留原模块路径，
   `grammar_loader` / `decompose/lines` / `cobol_parser` 的现有 import **一字不改**）。
2. **`parser/cobol_parser.py`**：
   - `_strip_cobol_line` → 薄委托 `CleanSource.strip_line`（或迁出后保留同名瘦包装）；
   - `_clean_lines` → 薄委托 `preprocess.line_stream.build(...).as_pairs()`，返回形态不变；
   - `parse()` 的 DIVISION/SECTION 定位、ws/linkage 切片**逻辑不动**，仅其用到的 `cobol_columns.*`
     经第 1 步重导出后透明指向相1。
3. **`decompose/lines.py`**：仍薄重导出，源可继续指 `parser.cobol_columns`（已透明转相1），**不必动**。

> 接线原则：**只加一层间接、不改任何判定/切分逻辑**。任一步若导致 43 单测产物有差异，即为搬错，回退。

## 8. 已确认的决定

- [包名] **`preprocess/`**（语义直白；否决 `phase1_preprocess/`/`pre/`）。✅
- [产物] 干净行流用 **`dataclass`（`CleanLine`/`CleanSource`）**，非裸 tuple；旧调用方经 `as_pairs()` 零改动适配
  （为相2 回溯打底）。✅
- [分刀顺序] **项1（本步=步骤15）保持零行为变化、只立相1 骨架与干净行流**，当回归基准；
  方言归一**不塞进第一刀**。✅（2026-06-25 用户定）
- [步骤16] **方言归一吸收**＝CRLF 归一 + `GO→GO TO` 归一，作相1 内**行为变更子步骤**，走前后产物比对。✅
- [COPY] **真·COPY 展开单独成步骤**；spike 桩生成（ProLeap 拐杖）**不并入主线**。✅

> 本步设计与上述决定经认可后，即按 §5 落地（仅落地项1：零行为变化的相1 骨架）。
