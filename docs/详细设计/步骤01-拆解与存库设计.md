## 3. 拆解规则（已确认）

| 编号 | 规则 |
|---|---|
| 注释/停用 | **指示列（第7列）= `*` 或 `/`** → 注释（**保留**）；**第1–6列 = `!!!!!!`** → 停用旧代码（**丢弃**）。 |
| ① | `PROGRAM-ID. ZPOLDWNM` → 类名 `ZPOLDWNM`。 |
| ② | `WORKING-STORAGE SECTION.`（L18–L1580）→ 参数定义块，命名 `ZPOLDWNMWSAA`（cob名+WSAA）。 |
| ③ | `COPY xxx`：`REC`/`SKM` 结尾≈实体类字段定义；`VARCOM`≈被调用的处理逻辑/方法；其余=other。**只分类登记，不展开解析其内容。** |
| ④ | `PROCEDURE DIVISION USING LETCMNT-PARAMS PMSPNT-PARAMS.`（L1588）→ 方法入口+入参。 |
| ⑤ | PROCEDURE DIVISION 内**每个有效 `SECTION`** → 一个文件（约 **133** 个，执行时以实际为准）。 |

### 3.1 行处理（去行号+变更标记，留代码区）— 已确认
对每条物理行按序判定：
1. 第1–6列 == `!!!!!!` → **丢弃**（停用旧代码）。
2. 否则指示列（第7列）∈ `* /` → **注释**：**保留**（注释含业务语义，对后续翻译有用）。
3. 否则取 **第7–72列**（指示列+代码区），`rstrip`，并去掉行尾 `<XXXXXX>` 变更标记。空行丢弃。

### 3.2 SECTION 边界
- SECTION 头 = **有效行**（非丢弃）匹配 `^\s*([A-Z0-9][A-Z0-9-]*)\s+SECTION\s*\.`。
- `!!!!!!` 开头的 SECTION 头（如 `!!!!!! 2080-CHECK-YBT SECTION.`、`!!!!!! B320-INCSUM-INF SECTION.`）**不识别为边界**（停用）。
- 排除 DATA/ENVIRONMENT/LINKAGE 下的 SECTION（CONFIGURATION/INPUT-OUTPUT/FILE/WORKING-STORAGE/LINKAGE）。
- 每个 SECTION 区间 = [本头行号, 下一头行号-1]；末段到 PROCEDURE 末尾。

## 4. 为什么新写脚本而非复用 `parser/cobol_parser.py`

现有 `parser/cobol_parser.py::parse()` 的 `_strip_cobol_line` 只处理 `* / D`，**不识别 `!!!!!!`**，
会把停用的 `!!!!!! ... SECTION.` 误当作有效边界（实测多切出 ~18 个停用段，总数 151 而非 133），
且它丢弃全部注释、不保留原始物理行号。本步对“`!!!!!!` 停用、保留注释、保留行号区间”有精确要求，
故**新建独立脚本** `scripts/decompose.py`，自带受控解析，不改动现有翻译器。

## 5. 代码设计（`scripts/decompose.py`）

### 5.1 数据结构
```python
@dataclass
class Block:
    program: str            # "ZPOLDWNM"
    source_file: str        # "ZPOLDWNM.cob"
    block_index: int        # META=0；WORKING-STORAGE=0；SECTION 从1递增（源码顺序）
    block_name: str         # "1000-INITIALISE" / "ZPOLDWNMWSAA" / "__META__"
    block_type: str         # "SECTION" | "WORKING-STORAGE" | "META"
    start_line: int         # 原文件物理起始行号(1-based)
    end_line: int           # 原文件物理结束行号
    raw_text: str           # 清洗后的块内容（= 文件内容）
    # 预留字段（本步留空，后续回填）
    params_in=None; params_out=None; calls=None; logic=None; java_code=None
```

### 5.2 函数清单（每个函数做什么 + 设计思路）

| 函数 | 签名 | 实现的功能 | 设计思路 |
|---|---|---|---|
| `read_raw_lines` | `(path)->list[str]` | 用 `latin-1` 读原文件、按物理行返回 | cob 含非 UTF-8 字节与定宽列，latin-1 保证“原样字节+列位可算” |
| `is_deactivated` | `(raw)->bool` | 第1–6列是否 `!!!!!!` | 规则 3.1-① |
| `is_comment` | `(raw)->bool` | 指示列是否 `* /` | 规则 3.1-② |
| `clean_line` | `(raw)->str` | 取第7–72列、去尾 `<...>`、rstrip | 规则 3.1-③ |
| `clean_block` | `(raw_lines,s,e)->str` | 对 [s,e] 逐行：丢 `!!!!!!`/空行，留注释与代码 | 块正文（文件与库 raw_text 一致） |
| `locate_divisions` | `(raw_lines)->dict` | 定位 WS/LINKAGE/PROCEDURE 起始行、USING 入参 | 仅在有效行上正则 |
| `extract_copies` | `(raw_lines,ws_s,ws_e)->list` | 收集 WS 内**有效** COPY 名 | 规则③数据来源 |
| `classify_copies` | `(names)->dict` | REC/SKM→rec_skm；含 VARCOM→varcom；其余 other | 规则③分类 |
| `extract_sections` | `(raw_lines,proc_start)->list[Block]` | 按 3.2 切出全部有效 SECTION 块 | 单趟扫描，记录原始行号区间 |
| `build_ws_block` | `(raw_lines,ws_s,ws_e)->Block` | 生成 `ZPOLDWNMWSAA` 参数块 | 规则② |
| `build_meta_block` | `(...)->Block` | 汇总类名、入参、COPY 分类、参数块名 | 规则①④ + ③汇总 |
| `slug_filename` | `(name,used)->str` | SECTION 名→唯一文件名 | 落盘去重 |
| `write_files` | `(blocks,meta,out_dir)` | 写各切片 + WSAA + manifest.json/md | 输出 A |
| `import_mongo` | `(manifest_path,uri,db,coll)` | 读 manifest→幂等 upsert 入库（**惰性 import pymongo**） | 输出 B，后补 |
| `main` | `()` | 解析参数→解析→写文件→（可选）入库→打印校验摘要 | 编排；`--mongo-uri` 可选 |

### 5.3 执行命令

## 6. MongoDB 写入设计（本步：文件优先，入库后补）

**环境现状（已探测）**：本机 27017 未监听、无 docker、项目解释器
`/data/models/llm-fa312/bin/python` **未安装 pymongo**、venv 禁止 `--user` 安装、connimongo MCP 不可用。
→ 故本步**只写文件**，入库延后。

**落地方式**：
- `manifest.json` 含**每块完整入库字段**（即 §6 文档 schema），作为“待入库数据源”。
- `--mongo-uri` 可选：不传=纯文件模式（零依赖，立即可跑）；Mongo 就绪后用 `--import-only --mongo-uri ...`
  读 manifest 完成导入。pymongo **惰性导入**，纯文件模式不依赖它。

**入库参数（待 Mongo 就绪由用户提供连接串）**：库 `cobol`，集合 `ZPOLDWNM`。

**文档 schema**：
```json
{
  "program":"ZPOLDWNM","source_file":"ZPOLDWNM.cob",
  "block_index":4,"block_name":"1000-INITIALISE","block_type":"SECTION",
  "start_line":1885,"end_line":2027,"raw_text":"...清洗后正文...",
  "params_in":null,"params_out":null,"calls":null,"logic":null,"java_code":null,
  "created_at":"...","updated_at":"..."
}
```
**幂等 upsert**，键=`{program, block_name}`：`$set` 写结构+原文+updated_at；
`$setOnInsert` 写 created_at 与预留空字段（避免重跑覆盖后续回填的 logic/java_code）。
唯一索引 `(program, block_name)`。文档构成：1×META + 1×WORKING-STORAGE + N×SECTION。
