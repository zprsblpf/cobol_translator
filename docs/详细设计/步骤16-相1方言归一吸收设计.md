## 1. 本步目标与非目标

**目标**：在相1 增加「方言归一」环节，对**有效代码行**做两项归一：
1. **CRLF 归一**；
2. **`GO <段名>` → `GO TO <段名>`** 归一（补回漏检的 GO TO）。

**非目标**：不动 COPY（真·展开单独步骤）；不动 segmenter/parser 的检测正则本身（仅靠喂给它们的行已归一，
使现有 `_GOTO_RE` 自然命中）；不改相1 已立的骨架契约（`CleanSource` 字段不变）。

---

## 2. 两项归一的性质判定（先讲清，再设计）

### 2.1 CRLF 归一 —— 已被吸收，本步只「明确归属」，不新增代码
现状 `columns.clean_line` 末尾已 `.rstrip("\n").rstrip("\r")`，干净行流里**早已无 CR**；
主线唯一读源点 `preprocess_file` 用 `readlines()` 逐行切分后即过 `clean_line`。故 CRLF 在相1
**已实际归一**。本步**不新增代码**，仅在文档明确「CRLF 归属相1 列处理」，并加一条断言性回归点防回潮。

### 2.2 `GO→GO TO` 归一 —— 真行为变更（本步实质内容）
主线 `_GOTO_RE = \bGO\s+TO\s+(…)` 只认 `GO TO`，省 TO 的 `GO 段名` 被漏检 → 这些段的 `go_tos`
缺失、`review_items` 少报、segmenter 也看不到该跳转。归一后被检出 = **行为变更（更正确）**。

## 3. 正本驱动的三层级联（关键修正：config 先行，代码不塞魔数）

> **2026-06-25 修正**：原设计把 GO→GO TO 正则写进 `dialect.py` 代码＝魔数，违背「config 正本驱动」。
> 按项目正本链 **`docs/翻译标准/`（用户源头）→ `config/specs/`（派生模板）→ 代码（只读应用）** 重排。

### 3.0 源头层：`docs/翻译标准/流程结构.md`（已起草，🔶待认可）
新建「流程结构（控制流）」分类标准，§一 GO TO 写明：本 shop `GO` 可省 `TO`、归一为 `GO TO`、
归一边界（GOBACK 不动/幂等/引号内不动）。**此层是源头**，config 与代码均从它派生。

### 3.1 模板层：`config/specs/segmentation_spec.yaml` 新增 `dialect_normalization` 节
把方言归一规则声明为机读正本（与既有 `paragraph_model.pattern` 把正则放 config 一致）：
```yaml
dialect_normalization:
  go_to:
    desc: 本 shop 省 TO 的 GO 段名 → 标准 GO TO（详见 docs/翻译标准/流程结构.md §一）
    pattern: '\bGO\s+(?!TO\b)(?=[0-9A-Za-z])'   # 命中省 TO 的 GO；排除 GO TO / GOBACK
    replacement: 'GO TO '
    quote_guard: true                            # 仅引号外施加（COBOL 语义，由代码保证）
```
新增方言只动本文件（数据驱动、不改代码），守「config 先行、实现跟随、不塞魔数」。

### 3.2 访问层：`config/grammar_loader.py` 新增访问器
仿 `verbs()`/`_label_re()` 加 `dialect_rules() -> list[dict]`（读 `_seg()["dialect_normalization"]`，
预编译 pattern），对外暴露方言归一规则。规则的「是什么」全在 config，访问层只做加载/编译。

### 3.3 实现层：新增 `preprocess/dialect.py`（只读 config 应用，**代码无方言正则**）
- **函数 `normalize(code: str) -> str`**：遍历 `grammar_loader.dialect_rules()`，对每条规则做替换；
  `quote_guard=true` 的规则**仅对引号外文本段**施加（按单/双引号扫描切分 code 为「引号内/外」交替段，
  只对引号外段 `re.sub`，再拼回）。
  - *职责分清*：**「方言是什么」（pattern/replacement）全来自 config**；代码只负责**「怎么安全套用」**
    （引号保护这一 COBOL 语义、遍历规则、拼接）——无任何方言正则字面量。
  - 延迟 `from config import grammar_loader`（仿 `columns.clean_line`，避免加载期环依赖）。

### 3.4 接线
`CleanSource.strip_line` 在 `columns.clean_line(...)` 之后追加 `return dialect.normalize(code)`
（停用/注释/调试分支仍直接返回 `""`，不进归一）。其余文件**零改动**——`build`/`as_pairs`/
`cobol_parser._strip_cobol_line`(薄委托)/segmenter 全部透明受益，parser 的 `_GOTO_RE` 自然命中。
