## 1. 本步目标与非目标

**目标**：新建 `asg/` 包（相2 产物），对外暴露三件：
1. **带类型节点模型**（dataclass 层级）：`Program/Section/Paragraph` 结构节点 +
   `IfStmt/EvaluateStmt/PerformStmt/GotoStmt/CallStmt/Leaf` 语句节点，取代裸 `Stmt` token 袋；
2. **引用已解析**：PERFORM/GO TO 目标 → 真实过程单元（THRU 区间一次算清）；数据名 → 变量定义(DDE)；
3. **visitor 基类**（`visit_<类名>` 分派约定）+ **一个单点 demo visitor**（GO TO → Java），
   证明「访问带类型节点」可替代 rules 的 token 嗅探。

**非目标（本步明确不做）**：
- **不动旧路径**：`parser/cobol_parser.py`、`translator/segmenter.py`、`translator/rules.py`
  **一行不改**。ASG 是旁路新产物，不接入旧 `build_skeleton`，不影响现有翻译输出。
- **不迁规则**：rules 的 MOVE/IF/PERFORM/CALL/STRING 翻译**不搬**（那是 §5 项3「逐类迁 visitor」的事，
  本步只用 GO TO **一个**动词做单点验证，证明通路）。
- **不建符号作用域的高级特性**：COBOL WS/LINKAGE 是程序级全局，本步符号表只做**程序级扁平查名**
  （大小写不敏感），嵌套/REDEFINES 视图等留后续（§7-Q）。
- **不碰 config 正本**：config/specs 不动；ASG 是中性 IR，翻译规范仍由相3 visitor 读 config（本步不涉及）。

## 2. `asg/` 包结构与各文件职责（瘦而专，单文件逻辑≈100 行内）

```
asg/                            ← 新建「相2」包（旁路，不接旧路径）
├── __init__.py                 瘦门面：导出节点类 + 基类 + 唯一入口 build_asg()
├── nodes.py                    节点模型（dataclass 层级：结构节点 + 语句节点）
├── registry.py                 过程注册表 ProcRegistry + 符号表 SymbolTable（引用解析的数据基座）
├── builder.py                  lift：复用 segmenter 的 Stmt → 提升为带类型节点 + 解析引用
└── visitor.py                  AsgVisitor 基类（visit_<类名> 分派）+ 单点 demo GotoJavaVisitor
```

调用关系（数据流，单向，旁路）：

```
CobolProgram(旧 parse 产物，只读)
       │
       ▼
asg.build_asg(program)                         ← 唯一入口（__init__ 门面）
   ├─ registry.build(program) ──► ProcRegistry（复用 _build_proc_order 四元）
   │                              SymbolTable（复用 program.working_storage/linkage_vars）
   └─ builder.build(program, registry)
        ├─ segmenter.split_paragraphs(section.lines)   ← 只读复用
        ├─ segmenter.segment(para_body) → list[Stmt]   ← 只读复用（裸树）
        └─ _lift(Stmt, registry) ──► 带类型节点（引用已解析）
                                          │
                                          ▼
                       Program 节点树（相2 产物 = 相3 唯一输入）
                                          │
                       AsgVisitor.visit(node)  ← 单点 demo：GotoJavaVisitor 验证
```

> **绞杀要点**：`builder` **调用**（不修改）`segmenter.segment` 取得已验证的确定性切分树，
> 再把裸 `Stmt` **提升**为带类型节点。token 嗅探（`tokens[0]=="GO"` 这类）**只在 `_lift` 集中发生一次**，
> 相3 visitor 从此只访问类型，不再嗅探——这正是要削的偶然复杂度（初步设计 §0）。

### 2.1 `asg/nodes.py`（节点模型）
- **设计思路**：用 `@dataclass` 平铺节点，**不做深继承**（COBOL 语句种类有限，继承徒增复杂）；
  visitor 用**类名分派**（`visit_IfStmt`），无需双分派样板。每节点带 `raw`/`lineno` 供回溯。
- **结构节点**：
  - `Program(program_id: str, sections: list[Section], registry: ProcRegistry)`——根节点，挂注册表。
  - `Section(name: str, paragraphs: list[Paragraph], lineno: int)`。
  - `Paragraph(label: str | None, stmts: list[Node], lineno: int)`——label=None 表段首无标号块。
- **语句节点**（带类型，取代裸 `Stmt`）：
  - `IfStmt(cond: list[str], then: list[Node], els: list[Node], raw, lineno)`。
  - `EvaluateStmt(subject: list[str], whens: list[tuple[list[str], list[Node]]], raw, lineno)`。
  - `PerformStmt(target: ProcRef | None, thru: ProcRef | None, inline_body: list[Node], header: list[str], raw, lineno)`
    ——`target`/`thru` 是**已解析**的过程引用（解析失败则 None + 保留 header，兜底）。
  - `GotoStmt(target: ProcRef | None, raw, lineno)`。
  - `CallStmt(name: str | None, using: list[str], raw, lineno)`——`name` 取自 `CALL 'XXX'`。
  - `Leaf(tokens: list[str], raw, lineno)`——**兜底**：本步未细分类型的简单语句（MOVE/COMPUTE/…）原样收，
    相3 visit 时落 `// TODO`（行为同今天保守，不臆测）。
- **`ProcRef`**：`ProcRef(name: str, unit: ProcUnit | None)`——引用解析结果的轻包装（名 + 解析到的单元）。

### 2.2 `asg/registry.py`（过程注册表 + 符号表）
- **`ProcUnit`**（dataclass）：一个可被 PERFORM/GO TO 的过程单元。
  字段：`name: str`（大写）、`kind: str`（section|paragraph）、`section: str`（所属 SECTION）、
  `order: int`（在 proc_order 中的序号，供 THRU 区间按序取）。
  *做什么*：proc_order 四元（步骤13）的**转正对象化**，替代后续临时建索引。
- **`ProcRegistry`**：
  - 构建：复用 `body_context._build_proc_order(program)`（只读调用）得四元列表 → 转 `list[ProcUnit]` + `dict[name→ProcUnit]`。
  - `resolve(name) -> ProcRef`：按名查单元（大小写不敏感），查不到 `unit=None`。
  - `resolve_thru(a, b) -> list[ProcUnit]`：按 `order` 取 [a..b] **闭区间**过程单元（THRU 区间一次算清，
    替代步骤12/13 在 rules 里的临时区间逻辑——本步只在 ASG 算清并挂到 `PerformStmt`，**不改 rules**）。
- **`SymbolTable`**：
  - 构建：复用 `program.working_storage + linkage_vars`（`Variable` 列表，只读），扁平展开为 `dict[name_upper→Variable]`。
  - `lookup(name) -> Variable | None`：数据名 → 定义项(DDE)，供相3 判型/前缀（本步先建表，单点 visitor 暂不用）。
  - *作用域决定（§9-Q3）*：COBOL WORKING-STORAGE/LINKAGE 是**程序级全局**，本步用**单层程序级扁平表**；
    GROUP 嵌套/REDEFINES 别名视图留后续（§7-Q1）。

### 2.3 `asg/builder.py`（lift：裸 Stmt → 带类型节点）
- **`build(program, registry) -> Program`**：遍历 `program.sections` → 每 SECTION
  `split_paragraphs(section.lines)`（复用）→ 每 paragraph `segment(body)`（复用）得裸 `Stmt` 列表 →
  逐个 `_lift` → 组装 `Paragraph/Section/Program`。
- **`_lift(st: Stmt, registry) -> Node`**：**唯一的 token 嗅探点**，按 `st.kind` + 首 token 分派：
  - `kind=="if"` → `IfStmt`（递归 lift then/else）；
  - `kind=="evaluate"` → `EvaluateStmt`（递归 lift 各 when 体）；
  - `kind=="perform"` → `PerformStmt`：从 header 提目标名/THRU/UNTIL，`registry.resolve`/`resolve_thru` 解析，递归 lift 内联体；
  - `kind=="simple"` 且首 token==`GO` → `GotoStmt`（提目标名，`registry.resolve`）；
  - `kind=="simple"` 且首 token==`CALL` → `CallStmt`（提 `'XXX'` 名 + USING）；
  - 其余 `simple` → `Leaf`（原样收 tokens/raw）。
  - *设计思路*：分派逻辑**抄自** rules `_skeleton_one`/`_sk_*` 的判定（复用既有正确判定），但产出**类型节点**而非 Java，
    且引用在此解析。这样 token 嗅探**收口一处**，相3 不再嗅探。

### 2.4 `asg/visitor.py`（visitor 基类 + 单点 demo）
- **`AsgVisitor`**：`visit(node)` → `getattr(self, "visit_"+type(node).__name__, self.generic_visit)(node)`；
  `generic_visit` 默认递归子节点。*分派约定（§9-Q1 决定）*：**`visit_<类名>` 命名分派**（仿 Python `ast.NodeVisitor`），
  不用双分派——样板少、加新节点只需加一个 `visit_X` 方法。
- **`GotoJavaVisitor(AsgVisitor)`**（**单点验证 demo**）：只实现 `visit_GotoStmt` → 产出与现 rules
  `_sk_control` 对 GO 的**等价 Java**（back-edge 状态机那一行），用于证明「访问 `GotoStmt.target`」
  可替代 rules `tokens[0].upper()=="GO"` 嗅探。*仅作通路验证，不接入主翻译，不改 rules*。

### 2.5 `asg/__init__.py`（瘦门面）
- 导出节点类、`ProcRegistry`/`SymbolTable`、`AsgVisitor`。
- **`build_asg(program: CobolProgram) -> Program`**：`registry.build` → `builder.build`，相2 对外**唯一入口**。
  *只做编排，不放逻辑*（瘦入口规范）。
